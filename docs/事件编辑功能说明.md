# 事件编辑功能说明

本文档依据当前工程中的类型定义（`src/types/event.ts`）、事件表单（`src/components/eventEditor/EventEditorForm.tsx`）与**预览运行时**（`src/utils/previewEventRuntime.ts`）整理，描述编辑器中的配置方式及各字段含义。若某触发类型在预览中尚未接入，文中会单独标明。

---

## 1. 事件数据模型每条事件（全局事件、楼层自定义事件、瓦片事件共用同一结构）对应一个 `GameEvent`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `eventId` | string | 事件唯一 ID；`nextEvent`、`event_triggered`、模板等会引用。 |
| `trigger` | `{ type, params? }` | 触发方式；`params` 为 JSON 对象，见下文「触发器」。 |
| `conditions` | 条件数组 | 可为空；空则视为恒通过。 |
| `conditionLogic` | `'and'` \| `'or'` | 多条件时为全部满足（AND）或任一满足（OR）。 |
| `conditionFailText` | string（可选） | 条件不满足时写入预览日志的提示文案。 |
| `actions` | 动作数组 | 条件通过后依次执行；每项可有 `delay`（毫秒，当前预览**不延时**，仅保存数据）。 |
| `nextEvent` | string（可选） | 执行完本事件动作后，继续查找并执行指定 `eventId` 的事件（同层全局/自定义/当前瓦片事件表）。 |

**编辑器操作简述**

- 触发器：下拉选择类型；仅 **碰撞 Tile**（`bump_tile`）额外提供「碰撞方向」下拉，其余触发器的 `params` 需在 JSON 中手写（若有）。
- 条件 / 动作：选择类型后，在 **「条件参数 (JSON)」** / **「动作参数 (JSON)」** 中编辑对象；语法错误失焦后会回退。
- 瓦片事件编辑面板可填 **后续事件 ID**（`nextEvent`）；全局/楼层事件面板以当前实现为准。

---

## 2. 比较运算符 `comparator`

在条件 `key_count`、`stat_require`、`monster_defeat_count` 中使用，字符串形式：

| 值 | 含义 |
|----|------|
| `>=` | 大于等于（默认值常用） |
| `<=` | 小于等于 |
| `>` | 大于 |
| `<` | 小于 |
| `==` 或 `=` | 等于 |

未识别运算符时，条件判定为不通过。

## 3. 触发器（Trigger）

以下为编辑器下拉中的全部 `trigger.type`。**UI 专项字段**仅 `bump_tile` 的 `direction`；其它 `params` 均为可选 JSON，按下列说明填写。

| type | 界面文案 | params（JSON） | 预览运行时 |
|------|----------|----------------|------------|
| `step_on_tile` | 玩家踩上 Tile | 无固定字段 | **已接入**：瓦片事件中，玩家走入该格后触发。 |
| `bump_tile` | 玩家碰撞 Tile | **`direction`**：`up` \| `down` \| `left` \| `right` \| `any`（编辑器下拉写入）；与玩家碰撞方向一致时才执行（`any` 不限制）。 | **已接入**：无法走入目标格时顶撞该格，对该格瓦片事件触发。 |
| `talk_to_tile` | 与 Tile 对话 | 无固定字段 | **已接入**：预览中空格/E 对话，且当前格为 NPC 时对瓦片事件触发。 |
| `use_item` | 使用道具 | 由策划约定，运行时未解析 | **预览未接入** |
| `defeat_monster` | 击败怪物 | 见下表「defeat_monster 参数」 | **已接入**：仅对本层 **全局事件 + 自定义事件** 扫描触发；**瓦片上**绑定的 `defeat_monster` 预览中**不会**因击败怪物而执行。 |
| `game_init` | 游戏初始化 | 无固定字段 | **已接入**：进入预览、初始化楼层时对本层全局+自定义事件触发。 |
| `stat_change` | 属性变化 | 由策划约定 | **预览未接入** |
| `time_trigger` | 时间触发 | 由策划约定 | **预览未接入** |
| `custom_event` | 自定义事件 | 由策划约定 | **预览未接入** |
| `key_change` | 钥匙数量变化 | 由策划约定 | **预览未接入** |
| `floor_change` | 楼层切换 | 由策划约定 | **预览未接入** |

### 3.1 `defeat_monster` 的 `trigger.params`

预览根据「击败发生坐标」过滤事件（见 `triggerMatchesDefeat`）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `matchAny` | boolean | 为 `true` 时，任意位置击败怪物都会匹配本事件。 |
| `x`, `y` | number | 指定格坐标；**同时省略** `x`、`y` 时视为任意击败（与未写坐标一致）。 |
| `floorId` | number（可选） | 楼层；省略则用当前击败所在层。 |

示例：

```json
{ "x": 3, "y": 5, "floorId": 1 }
```

```json
{ "matchAny": true }
```

## 4. 条件（Condition）

`params` 均为 JSON 对象。下列为预览中**已实现**的判定逻辑；未列出的 `type` 在预览里按 **恒为 true** 处理（`default` 分支）。

| type | 界面文案 | params | 说明 |
|------|----------|--------|------|
| `key_count` | 钥匙数量 | `color`：`yellow` \| `blue` \| `red`（默认 `yellow`）<br>`count`：number（默认 `0`）<br>`comparator`：见 §2（默认 `>=`） | 与玩家当前对应颜色钥匙数量比较。**绿色钥匙未参与预览玩家属性**，勿依赖 `green`。 |
| `stat_require` | 属性要求 | `stat`：`hp` \| `attack` \| `defense` \| `gold`（默认 `hp`）<br>`value`：number（默认 `0`）<br>`comparator`：见 §2（默认 `>=`） | 其它 `stat` 当前按 `0` 比较。 |
| `current_floor` | 当前楼层 | `floorId` 或 `floor`：number | `ctx.floorId` 与该值相等为真。 |
| `event_triggered` | 事件触发状态 | `eventId`：string<br>`status`：`triggered`（默认）或 `not_triggered` | 检查本次预览会话中是否已执行过该 `eventId`（已写入 `triggeredEventIds`）。 |
| `has_item` | 拥有道具 | `itemType`：`yellowkey` \| `bluekey` \| `redkey` | 仅三种钥匙；对应数量 &gt; 0。其它值恒为 false。 |
| `custom_variable` | 自定义变量 | `key`：string<br>`value`：任意 | `ctx.customVars[key] === value`（严格相等）。 |
| `random_probability` | 随机概率 | `probability` 或 `p`：0～1 的 number（默认 `0.5`） | `Math.random() < prob`；每次判定重新随机。 |
| `monster_defeat_count` | 怪物击败数量 | `count`：number（默认 `0`）<br>`comparator`：见 §2（默认 `>=`）<br>`floorId`：number（可选） | 统计 **当前预览会话** 内，已击败怪物格数量；格键格式为 `` `${floorId}:${x}:${y}` ``，省略 `floorId` 时用当前层。 |
| `monsters_at_defeated` | 指定格怪物已击败 | `floorId`：number（可选，默认当前层）<br>`positions`：`[{ x, y }, ...]` |所列坐标格**全部**在已击败集合中则为真；`positions` 为空数组时恒为真。 |

## 5. 动作（Action）

每项动作可含 **`delay`**（毫秒）：当前预览实现**不等待延时**，动作仍立即执行。

| type | 界面文案 | params | 预览行为摘要 |
|------|----------|--------|----------------|
| `show_hint` | 显示提示 | `text` 或 `message`：string | 文本写入预览日志。 |
| `show_dialog` | 显示对话 | 同上 | 与 `show_hint` 相同，仅日志（无头像/UI）。`avatar` 等字段可存 JSON，预览不展示。 |
| `change_stat` | 改变属性 | `stat`：`hp` \| `attack` \| `defense` \| `gold`（默认 `hp`）<br>`operation`：`+` \| `add` \| `-` \| `subtract`（默认 `+`）<br>`value`：number（默认 `0`） | 加减玩家属性；未列出的 `stat` 不修改。 |
| `get_item` | 获得道具 | `itemType`：string<br>`count`：number（默认 `1`） | 支持的 `itemType`：`yellowkey`、`bluekey`、`redkey`、`hp`、`hplarge`、`attackgem`、`defencegem`、`luckycoins`、`gold`。 |
| `consume_key` | 消耗钥匙 | `color`：`yellow` \| `blue` \| `red`（默认 `yellow`）<br>`count`：number（默认 `1`） | 对应钥匙扣减，不低于0。**无 green**。 |
| `consume_item` | 消耗道具 | 任意 | **预览无效果**（空实现）。 |
| `teleport_player` | 传送玩家 | `x` 或 `targetX`、`y` 或 `targetY`：number（默认当前坐标）<br>`floorId`：number（可选，默认当前层） | 修改工作层与玩家坐标，并打日志。 |
| `change_floor` | 切换楼层 | `floorId` 或 `floor`：number | 若地图中存在该层，切换到该层并将玩家置于该层 `playerStart`。 |
| `change_tile` | 改变 Tile | `targetX` 或 `x`、`targetY` 或 `y`：number（默认玩家当前格）<br>`floorId`：number（可选，默认当前层）<br>`newTileType`：string（默认 `floor`） | 在覆盖层生成新瓦片类型（需能在 `presetTiles` 中解析）；失败则该格置为清除覆盖（`null`）。 |
| `trigger_custom_event` | 触发自定义事件 | `eventId`：string | 在本层全局、自定义、**当前玩家脚下瓦片**的 `events` 中查找并**同步执行**该事件（可链式嵌套）。 |
| `set_custom_variable` | 设置自定义变量 | `key`：string<br>`value`：任意 | 写入 `customVars`。 |
| `game_victory` | 游戏胜利 | `text`：string（可选） | 日志提示，无真正结束流程。 |
| `game_failure` | 游戏失败 | `text`：string（可选） | 同上。 |
| `lock_player` | 锁定玩家 | — | **预览无效果**。 |
| `unlock_player` | 解锁玩家 | — | **预览无效果**。 |
| `play_sound` | 播放音效 | — | **预览无效果**。 |
| `shake_screen` | 震动屏幕 | — | **预览无效果**。 |
| `no_op` | 无操作 | — | 无效果。 |

## 6. 预览中与事件相关的其它规则

- **已击败怪物集合**：键为 `` `${floorId}:${x}:${y}` ``（与条件 `monsters_at_defeated`、`monster_defeat_count` 一致）；击败判定以预览内战斗逻辑为准（攻击力大于怪防御等）。
- **`change_tile` 的坐标键**：与上款相同格式，用于 `tileOverrides`。
- **条件失败**：若配置了 `conditionFailText`，会先写入日志再跳过动作；不会将本事件记入「已触发」集合（因内部在判定失败时提前 `return`）。
- **导出 JSON**：事件数据随地图保存；实际游戏引擎是否支持全部触发器/动作，以目标运行时为准。
