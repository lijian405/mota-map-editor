# 剧情系统设计（运行时 + 编辑器规格）

本文描述当前工程内剧情系统的**数据格式、触发规则、执行语义与存档**，供策划配置与后续**剧情编辑器**（可视化编辑 `story.json`、校验、预览）对齐实现。

**权威数据源**：[`map_data/story.json`](map_data/story.json)（单文件，根对象含 `stories` 数组）。

**运行时入口**：Autoload [`StoryController`](scripts/story/story_controller.gd) → [`StoryRegistry`](scripts/story/story_registry.gd) 索引触发器 → [`story_command_builder.gd`](scripts/story/story_command_builder.gd) 将 `actions` 转为指令 → [`ScriptRunner`](scripts/script/script_runner.gd) 顺序执行。

---

## 1. 设计目标

- **触发器 + 动作组**：满足条件时进入剧情流程；流程中**玩家不可移动**（`GameManager.cutscene_playing`）。
- **数据驱动**：剧情以 JSON 描述，便于编辑器读写、版本管理、热修。
- **与旧数据共存（可选）**：通过 `runEvent` 引用 [`map_data/event.json`](map_data/event.json) 中的事件定义，逐步迁移到纯 `actions`。

---

## 2. 根文件结构

```json
{
  "stories": [ /* Story 对象数组 */ ]
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `stories` | `Array` | 多条剧情；每条为 **Story 对象**（见下节）。 |

---

## 3. Story 对象

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | `String` \| `Number` | 建议必填 | 剧情唯一标识；存档用 **字符串**（运行时 `str(id)`）。编辑器应规范为字符串避免歧义。 |
| `desc` | `String` | 否 | 策划备注，运行时忽略。 |
| `repeatable` | `Boolean` | 否 | 默认 `false`。`false` 时完成后写入 `story_completed`，同存档不再触发。 |
| `order` | `Number` | 否 | 默认 `0`。同一触发键下多条剧情时**先按 `order` 升序，再按 `id` 字符串升序**。 |
| `trigger` | `Object` | 是 | 触发器；见第 4 节。 |
| `actions` | `Array` | 是 | 动作列表，**按顺序**执行；见第 5 节。 |

**编辑器校验建议**：

- `id` 全局唯一（或与团队约定命名空间，如 `npc_3`、`tile_1_1_event1`）。
- `trigger` 与 `actions` 非空；`actions` 可为空数组（运行时该剧情会被跳过，无表现）。

---

## 4. 触发器（`trigger`）

所有触发器均为对象，字段 `type` 区分种类。

### 4.1 通用约定

- 触发时由 `StoryController` 将命中剧情入队，与 `ScriptRunner` **串行**执行；执行期间不响应新的触发入队（`runner.is_running()` 时直接返回）。
- **进层**（`floor_changed`）：匹配 `floor_enter` 的剧情全部入队后依次执行。
- **踩格**（`player_moved`）：匹配当前楼层 + 当前格子的 `tile_enter` 剧情入队执行。
- **NPC 对话**（`npc_interact`）：匹配 `npc_interact` 的剧情入队执行；**无匹配则无任何对话**（无默认 NPC 表）。
- **击败怪物**（`monsters_defeated`）：在指定楼层，当列出格子上怪物被击败后判定（见 4.5）；击败时由 `GameManager.monster_defeated` 驱动。

### 4.2 `floor_enter` — 进入楼层

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | 固定 `"floor_enter"` | |
| `floor` | `Number` | 楼层 ID，与地图 `floorId` 一致。 |

### 4.3 `tile_enter` — 进入格子

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | 固定 `"tile_enter"` | |
| `floor` | `Number` | 所在楼层。 |
| `tile` | `String` | 格式 **`"x,y"`**（与地图网格一致，如 `"1,1"`）。 |

### 4.4 `npc_interact` — 与 NPC 交互

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | 固定 `"npc_interact"` | |
| `npc` | `String` | 与地图上 NPC 的 `npc_id`（来自瓦片 `name`）一致。 |
| `floor` | `Number` | **可选**。若省略：任意楼层与该 NPC 对话均可触发（索引为「仅 NPC」）。若填写：仅在该层触发；可与「无 floor」条目**同时存在**，运行时**合并**后按 `order`、`id` 排序。 |
| `requiresStories` | `Array` | 可选。所有列出的 `story id` 必须已完成（`repeatable:false` 的剧情会写入 `story_completed`）才会触发该剧情。用于多段 NPC 对话。 |
| `forbidStories` | `Array` | 可选。若任意列出的 `story id` 已完成，则该剧情不会触发。 |

### 4.5 `monsters_defeated` — 击败指定格子上的怪物

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | 固定 `"monsters_defeated"` | |
| `floor` | `Number` | **必填**。与 `floorId` 一致。 |
| `positions` | `Array` | 与 `tiles` **二选一**（等价）。每项可为：`"x,y"` 字符串、`{ "x", "y" }`、或地图**线性下标**整数（需当前 `MapLoader` 与地图宽度一致）。 |
| `tiles` | `Array` | 同 `positions`。 |
| `match` | `String` | 可选，默认 `"all"`。`all`：**每一个**列出格子都必须发生过击败；`any`：**至少一个**列出格子发生过击败即满足。 |
| `monsterIds` | `Array` | 可选。与 `positions` **按下标一一对应**；非空字符串时，该格必须曾击败 **tile `name` 与该字符串一致**的怪（与地图数据里怪物瓦片名一致，如 `"100"`）。未写或长度不足的下标不校验种类。 |

击败记录写入存档（`monster_spots_cleared` + `monster_spots_monster_key`），切层再回来仍视为已清；`repeatable: false` 的剧情触发后写入 `story_completed`，不会重复播。

**示例**（第 1 层击败 `(3,4)` 与 `(5,6)` 后触发）：

```json
"trigger": {
  "type": "monsters_defeated",
  "floor": 1,
  "positions": ["3,4", "5,6"],
  "match": "all"
}
```

## 5. 动作（`actions`）

每条动作为一个对象，必须包含 `type`（字符串，大小写敏感，与下表一致）。

### 5.1 动作级 `delay`（可选）

任意动作可带 **`delay`**（`Number`，单位 **毫秒**）。

**语义**：在执行该动作**之前**先等待 `delay` ms（由运行时插入 `wait_ms` 指令）。

### 5.2 已实现动作一览

| `type` | 说明 | 主要字段 | 运行时备注 |
| --- | --- | --- | --- |
| `chat` | 对话 | 见 5.3 | 多句多角色会拆成多条内部 `chat` 指令。 |
| `playSound` | 播放音效 | `sound`：路径，如 `sound/level3Event.mp3` 或相对 `res://` | 无文件时警告；不阻塞等待播放结束。 |
| `screenFade` | 中间面板区域淡入淡出 | `animationType`：`fadeIn` / `fadeOut`（含 `out`/`in` 即判定）；`duration`：秒（可选，默认用 [`scene_fader.gd`](scripts/ui/scene_fader.gd) 中 `duration`） | 遮罩挂在 [`middle_panel.tscn`](scens/middle_panel.tscn) 的 `PanelFade`（`Polygon2D`），非全屏。 |
| `appear` | 在格子生成实体 | `animationType`（可选）；`entities` 数组，见 5.4 | 映射为 `appear_entities`。 |
| `move` | 寻路移动（A*） | `entity`，`from`（可选），`to`，`stepMs`（可选），见 5.6 | 播放逐格行走过程。 |
| `disappear` | 批量移除实体/墙 | `ops`，见 5.7 | 基于线性下标；会调用 `record_removed_object`。 |
| `removeTile` | 移除指定格子上的图块/实体 | `pos`，可选 `kind` / `name`，见 5.8 | 会调用 `record_removed_object`，切层后也不再出现。 |
| `changeFloor` | 切换楼层 | `floor`：目标楼层 ID | 调用 `MapLoader.load_floor` + `GameManager.init_floor`。 |
| `changePlayerState` | 修改玩家数值 | 推荐 `changes`：`{ "hp": n, "gold": n, ... }`；或直接平铺数值字段（会剔除保留键） | 数值型为 **累加**到当前 `player_state`。 |
| `giveItem` | 给予物品 | `items` 或 `itemIds`：字符串数组，与 [`prop.json`](map_data/prop.json) 条目名一致 | 走 `GameManager.grant_item_by_name`。 |
| `runEvent` | 内嵌旧事件 | `eventId` 或 `id`：[`event.json`](map_data/event.json) 中的键 | 在 [`story_controller.gd`](scripts/story/story_controller.gd) 的 `_resolve_story_commands` 中**展开**为 `event_to_commands`；若 `has_event` 已 true 则**不展开**该段。 |

### 5.3 `chat` — 对话内容

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | `"chat"` | |
| `content` | `Array` | 每元素为一「句/页」： |

**`content` 元素格式**（二选一）：

1. **对象**（推荐，支持角色名）  
   - `name`：`String`，显示在姓名栏。  
   - `content`：`String` **或** `Array` of `String`（多行合并为一页，`\\n` 连接）。

2. **字符串**  
   - 整句文本；姓名为空。

### 5.4 `appear` — `entities` 项

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | `"player"` \| `"npc"` \| `"monster"` \| `"item"` | |
| `pos` | `Number` \| `"x,y"` \| `{ "x", "y" }` | **线性下标**（按当前层 `mapWidth` 换算格子）或显式坐标。 |
| `name` | `String` | `monster`：怪物图鉴 ID；`npc`：NPC 瓦片名，默认可 `npc1`；`item`：道具 ID（对应 `images/items/<id>.png` / `prop.json` 键）。 |
| `delay` | `Number` | 可选，毫秒；在该实体生成**前**等待。 |

### 5.5 已预留、当前为占位（`custom`）

以下 `type` 在 [`story_command_builder.gd`](scripts/story/story_command_builder.gd) 中会变为 `custom`，**运行时无逻辑**（可后续在 `ScriptRunner` 实现）：

- `playTileAnimation` — 图块上动画  
- `spawnTile` / `spwanTile`（文档笔误兼容）— 指定资源铺格  

编辑器可列为「计划中」，或生成时标灰。

---

### 5.6 `move` — A* 寻路移动（播放过程）

将 player 或指定 NPC 从起点移动到终点，基于当前层网格做 4 方向 A*；遇到 `is_passable == false` 的格子视为阻挡。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | 固定 `"move"` | |
| `entity` | `Object` \| `String` | `"player"` 或 `{ "type": "npc", "npcId": "wise" }`（也兼容字段 `id`）。 |
| `from` | `Number` \| `"x,y"` \| `{ "x", "y" }` | 可选；不填则用实体当前格子；填写则先把实体放到该格再开始走。 |
| `to` | `Number` \| `"x,y"` \| `{ "x", "y" }` | 目标位置（必填）。 |
| `stepMs` | `Number` | 可选，默认 `120`；每走一格的动画时长（毫秒）。 |

**示例**（让玩家从 `(1,1)` 走到 `(5,1)`）：

```json
{
  "type": "move",
  "entity": "player",
  "from": "1,1",
  "to": "5,1",
  "stepMs": 120
}
```

### 5.7 `disappear` — 批量移除实体/墙（线性下标）

用于批量移除当前层指定位置的对象；参数使用**线性下标**（按当前层 `mapWidth` 换算格子）。会调用 `record_removed_object`，因此切层回来也不会刷回。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | 固定 `"disappear"` | |
| `ops` | `Array` | 每个元素是一个对象，key 为 kind，value 为线性下标数组。 |

**支持的 kind**（与运行时一致）：

- `npc`：移除 NPC（`tile.type == "npc"`）
- `monster`：移除怪物（`tile.type == "monster"`）
- `wall`：移除墙（`tile.type == "terrain"` 且 `tile.name == "wall"`）

**示例**（移除下标 10、11 的墙，以及下标 42 的怪物）：

```json
{
  "type": "disappear",
  "ops": [
    { "wall": [10, 11] },
    { "monster": [42] }
  ]
}
```

### 5.8 `removeTile` — 移除指定位置图块/实体

移除当前层某个格子上的对象；可用 `kind`（`tile.type`）或 `name`（`tile.name`）做过滤。不写过滤条件时会移除该格上所有可匹配对象（不含玩家）。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | 固定 `"removeTile"` | |
| `pos` | `Number` \| `"x,y"` \| `{ "x", "y" }` | 目标格子；支持线性下标或坐标。 |
| `kind` | `String` | 可选；匹配 `get_tile_type()`（如 `npc` / `monster` / `item` / `terrain` / `gate`）。 |
| `name` | `String` | 可选；匹配 `get_tile_name()`（如 `wall` / `stairs_down` / `100` / `yellowkey`）。 |

**示例**（移除 `(3,4)` 的怪物）：

```json
{
  "type": "removeTile",
  "pos": "3,4",
  "kind": "monster"
}
```

## 6. 与 `event.json` 的关系（`runEvent`）

- 剧情仍可使用 **`runEvent`** 复用旧事件步骤，避免一次性重写大体量 cutscene。  
- 展开时会对 **`eventId` 写入 `event_triggered`**（与直接 `play_event` 一致），与 **`story_completed`** 独立。  
- **编辑器**：可提供「导入 event 为只读预览」或「一键展开为 actions」（离线工具），减少双源维护。

## 7. 存档与标记

[`EventFlagsStore`](scripts/flags/event_flags_store.gd) 持久化字段（经 `GameManager` 的 `world.flags`）：

| 键 | 含义 |
| --- | --- |
| `story_completed` | `story id` → 已完成（`repeatable == false` 时写入）。 |
| `event_triggered` | `event id` → 旧事件已触发（含 `runEvent` 展开、`play_event`）。 |
| `npc_talked` | 遗留字段，当前逻辑未使用，兼容旧存档。 |
| `monster_spots_cleared` | 键 `floor|x,y` → 该格已发生过怪物击败。 |
| `monster_spots_monster_key` | 同上键 → 被击败怪的 tile `name`（供 `monsterIds` 校验）。 |

## 8. 剧情编辑器功能建议（产品向）

以下为与当前实现对齐的**建议能力清单**，便于分阶段开发编辑器。

1. **工程**  
   - 读写单文件 `map_data/story.json`；格式化 JSON；Git 友好 diff。  

2. **剧情列表**  
   - 表格：id、desc、trigger 摘要、`repeatable`、`order`。  
   - 筛选：按触发类型、楼层、NPC id、格子。  

3. **触发器编辑**  
   - 下拉 `type`；动态表单（floor、tile、npc、可选 floor）。  
   - **冲突检测**：同一 `(type, floor, tile)` 或 `(npc, floor?)` 多条时提示，依赖 `order` 排序。  

4. **动作编辑**  
   - 按 `type` 切换表单；`delay` 统一控件。  
   - `chat`：可视化多行角色台词；预览导出 JSON。  
   - `appear`：实体列表增删；坐标可选「地图点选」（需读 `map_data`）。  

5. **校验**  
   - JSON Schema 或自定义规则：必填字段、`tile` 格式、`sound` 文件存在性、`eventId` / 怪物 ID / prop 名引用存在性。  

6. **调试**  
   - 运行时 [`ScriptRunner.debug_log_actions`](scripts/script/script_runner.gd) 打印每条指令；编辑器可说明如何开关。  

7. **迁移工具（可选）**  
   - `event.json` → 生成带 `runEvent` 的 Story；或半展开为 `actions`。

## 10. 附录：原始需求表（概念对齐）

以下为最初功能规划与实现的对应关系，供对照；**以正文 JSON 字段为准**。

**触发器**

| 类型 | 描述 | 参数 |
| --- | --- | --- |
| floor_enter | 进入指定楼层 | `floor` |
