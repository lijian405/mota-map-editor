# 魔塔Tile地图编辑器设计方案

## 一、项目概述

开发一款**可视化魔塔Tile地图编辑器**，支持魔塔核心元素（地形、道具、怪物、NPC、机关等）的拖拽式编辑，最终导出标准JSON格式关卡数据，供游戏引擎直接加载。目标是降低关卡设计门槛，提升魔塔关卡创作效率。

## 二、核心功能模块

### 1. 地图画布管理

- **网格系统**：支持自定义地图宽高（如20×20、30×30），显示/隐藏网格线，网格尺寸自适应缩放。每个网格可以放置一个Tile。像素为32\*32。
- **视图控制**：鼠标滚轮缩放画布、拖拽平移画布，支持1:1视图重置。
- **图层管理**：
  - 底层：地形层（墙、地板、楼梯等）
  - 上层：物体层（道具、怪物、NPC、门等）
  - 事件层：隐藏的触发事件（如踩踏板、传送点）

### 2. Tile库管理

- **预设Tile分类**：
  | 分类  | 包含元素                           |
  | --- | ------------------------------ |
  | 地形  | 墙（不可破坏）、地板（可通行）、楼梯（上/下）、传送点    |
  | 道具  | 血瓶（小/大）、攻击宝石、防御宝石、金币、钥匙（黄/蓝/红） |
  | 门   | 黄门、蓝门、红门（需对应钥匙开启）              |
  | 怪物  | 史莱姆、蝙蝠、骷髅等（含属性面板）              |
  | NPC | 商人、老人（含对话文本编辑）                 |
  | 机关  | 陷阱（扣血）、隐藏墙（可破坏）、触发式道具          |
- **自定义Tile**：支持导入图片作为新Tile，设置碰撞属性和类型标签。

### 3. 地图编辑功能

- **基础操作**：左键放置Tile、右键删除Tile、拖拽连续填充。
- **高级操作**：
  - 区域选择（框选）、复制/粘贴/剪切选中区域。
  - 撤销/重做（支持20步历史记录）。
  - 批量替换（将某类Tile统一替换为另一类）。
- **属性编辑**：
  - 选中Tile后，右侧面板显示可编辑属性（如怪物的血量、攻击、防御，NPC的对话内容）。
  - 支持预设怪物模板（如“史莱姆模板”“骷髅模板”），快速应用属性。

### 4. 数据导入导出

- **导出JSON**：生成标准关卡数据文件，包含地图尺寸、Tile列表、属性配置、玩家初始状态。
- **导入JSON**：加载已有JSON文件，继续编辑关卡。
- **多楼层管理**：支持创建/切换楼层（如1层、2层…），每层独立存储数据，导出时包含楼层关联关系。

### 5. 预览与验证

- **简易预览**：在编辑器内模拟角色移动（WASD控制），测试碰撞检测、道具拾取、门开启逻辑。
- **关卡验证**：自动检查逻辑错误（如“无楼梯”“怪物属性为负”“钥匙数量不足以开门”），并给出提示。

## 四、技术栈选型

| 模块    | 技术选型                                        |
| ----- | ------------------------------------------- |
| 前端框架  | React 18 + TypeScript（类型安全，组件化开发）           |
| 画布渲染  | PixiJS（高性能2D渲染，支持Tile地图缩放、平移）               |
| 状态管理  | Redux Toolkit（管理地图数据、历史记录、选中状态）             |
| UI组件库 | Ant Design（快速搭建工具栏、属性面板、弹窗）                 |
| 文件操作  | FileSaver.js（导出JSON）、FileReader API（导入JSON） |
| 桌面打包  | Electron（可选，将Web应用打包为Windows/macOS桌面程序）     |
| 构建工具  | Vite（开发热更新，构建速度快）                           |

## 五、界面设计思路

1. **顶部工具栏**：新建/打开/保存地图、撤销/重做、缩放控制、预览按钮、多楼层切换。
2. **左侧Tile库**：分类标签页（地形/道具/怪物等），Tile图标网格展示，支持搜索。
3. **中间画布区**：显示当前楼层地图，网格背景，鼠标悬停显示Tile坐标和类型。
4. **右侧属性面板**：
   - 未选中Tile：显示地图整体信息（宽高、楼层数）。
   - 选中Tile：显示类型、坐标、可编辑属性（如怪物血量输入框、NPC对话文本域）。
5. **底部状态栏**：显示鼠标坐标、当前操作提示、验证结果（如“关卡无逻辑错误”）。

## 六、核心开发流程

1. **基础框架搭建**：初始化React+Vite项目，集成PixiJS和Ant Design，实现画布网格渲染。
2. **Tile库与放置功能**：开发预设Tile组件，实现左键放置、右键删除逻辑。
3. **属性编辑与数据管理**：设计Redux状态结构，实现属性面板数据绑定，支持撤销/重做。
4. **多楼层与导入导出**：开发楼层切换功能，实现JSON文件的读写。
5. **预览与验证**：添加简易游戏逻辑（角色移动、碰撞检测），开发关卡验证工具。
6. **优化与打包**：性能优化（如Tile懒加载），Electron打包（可选）。

# 魔塔Tile地图编辑器 - 事件系统细化方案

## 一、事件系统核心概念

事件系统是魔塔关卡的“灵魂”，负责处理**玩家与地图元素的交互逻辑**（如踩踏板开门、与NPC对话获得道具、击败怪物解锁隐藏区域等）。本系统采用\*\*“触发条件 → 执行条件 → 动作序列 → 后续事件”\*\*的链式结构，支持可视化编辑，最终导出为游戏引擎可解析的JSON数据。

## 二、事件分类与组成

### 1. 事件绑定对象

事件可绑定到以下三类对象：

- **Tile事件**：绑定到具体地图格子（如踩地板触发、与NPC对话触发）。
- **全局事件**：不绑定具体Tile，满足条件时自动触发（如“玩家攻击力≥100时自动获得隐藏道具”）。
- **自定义事件**：需通过其他事件手动触发（如“击败Boss后触发‘通关动画’自定义事件”）。

### 2. 事件核心组成

每个事件由以下四部分组成：

| 组成部分     | 说明                            | 示例                                   |
| -------- | ----------------------------- | ------------------------------------ |
| **触发条件** | 事件开始的“导火索”，即玩家做了什么操作触发该事件。    | 玩家踩上Tile、与Tile对话、使用道具、击败特定怪物、游戏初始化时。 |
| **执行条件** | 事件触发后，需满足的“前置要求”，不满足则无法执行动作。  | 玩家拥有黄钥匙×1、攻击力≥50、未触发过该事件、击败过史莱姆×3。   |
| **动作序列** | 条件满足后，按顺序执行的“具体操作”。           | 获得血瓶×1、攻击力+10、开启红门、传送至(5,5)、显示对话文本。  |
| **后续事件** | 动作执行完毕后，自动触发的“下一个事件”（支持链式触发）。 | 触发“获得道具”事件后，自动触发“隐藏墙消失”事件。           |

## 三、触发条件（Trigger）详解

支持以下10类常用触发条件，可通过编辑器“勾选+参数配置”快速设置：

| 触发条件类型       | 说明                                  | 配置参数示例                             |
| ------------ | ----------------------------------- | ---------------------------------- |
| **玩家踩上Tile** | 玩家移动到绑定事件的Tile时触发（最常用）。             | 无（绑定Tile后自动生效）。                    |
| **与Tile对话**  | 玩家面向Tile按空格键/回车键时触发（适用于NPC、告示牌）。    | 无。                                 |
| **使用道具**     | 玩家在背包中使用特定道具时触发（如使用“炸弹”炸墙）。         | 道具类型（如“炸弹”）。                       |
| **击败特定怪物**   | 玩家击败绑定事件的怪物Tile时触发。                 | 无（绑定怪物Tile后自动生效）。                  |
| **游戏初始化**    | 关卡加载完成时自动触发（适用于设置初始状态、显示开场对话）。      | 无。                                 |
| **属性变化**     | 玩家某属性（HP/攻击/防御/金币）达到/超过/低于特定值时触发。   | 属性类型（HP）、比较符（≥）、值（1000）。           |
| **时间触发**     | 玩家在当前楼层停留特定时间后触发（如“停留30秒后陷阱激活”）。    | 时间（秒，如30）。                         |
| **自定义事件触发**  | 被其他事件的“后续事件”调用时触发（用于构建复杂逻辑链）。       | 自定义事件ID（如“event\_boss\_defeated”）。 |
| **钥匙数量变化**   | 玩家获得/使用特定颜色钥匙时触发。                   | 钥匙颜色（黄/蓝/红）、变化类型（获得/使用）。           |
| **楼层切换**     | 玩家进入/离开当前楼层时触发（如“离开1层时显示‘2层危险’提示”）。 | 切换类型（进入/离开）、楼层ID（如2）。              |

## 四、执行条件（Condition）详解

支持以下8类执行条件，可**多条件组合**（“与”/“或”逻辑），并设置“条件不满足时的提示文本”：

| 执行条件类型     | 说明                                   | 配置参数示例                           |
| ---------- | ------------------------------------ | -------------------------------- |
| **拥有道具**   | 玩家背包中存在特定道具（如“拥有炸弹”）。                | 道具类型（炸弹）、数量（≥1）。                 |
| **属性要求**   | 玩家某属性满足特定要求（如“攻击力≥50”）。              | 属性类型（攻击）、比较符（≥）、值（50）。           |
| **钥匙数量**   | 玩家拥有特定颜色钥匙的数量（如“黄钥匙≥2”）。             | 钥匙颜色（黄）、比较符（≥）、值（2）。             |
| **事件触发状态** | 某事件是否已触发过（如“未触发过‘获得红宝石’事件”）。         | 事件ID（如“event\_ruby\_1”）、状态（未触发）。 |
| **怪物击败数量** | 特定怪物被击败的总数量（如“击败史莱姆≥5”）。             | 怪物ID（如“slime”）、比较符（≥）、值（5）。      |
| **当前楼层**   | 玩家是否在特定楼层（如“在3层时”）。                  | 楼层ID（如3）。                        |
| **自定义变量**  | 自定义变量满足特定值（如“变量‘boss\_awake’=true”）。 | 变量名（boss\_awake）、比较符（=）、值（true）。 |
| **随机概率**   | 按概率触发（如“30%概率获得额外金币”）。               | 概率（%，如30）。                       |

## 五、动作序列（Action）详解

支持以下15类动作，可**按顺序排列**，并设置“动作执行延迟”（如“显示对话后延迟2秒再传送”）：

| 动作类型          | 说明                                      | 配置参数示例                             |
| ------------- | --------------------------------------- | ---------------------------------- |
| **获得道具**      | 玩家获得指定道具（如“获得大血瓶×1”）。                   | 道具类型（大血瓶）、数量（1）。                   |
| **改变属性**      | 玩家属性增加/减少/设置为特定值（如“攻击力+10”“HP设置为1000”）。 | 属性类型（攻击）、操作类型（+）、值（10）。            |
| **消耗道具**      | 玩家背包中移除指定道具（如“消耗炸弹×1”）。                 | 道具类型（炸弹）、数量（1）。                    |
| **消耗钥匙**      | 玩家消耗特定颜色钥匙（如“消耗黄钥匙×1”）。                 | 钥匙颜色（黄）、数量（1）。                     |
| **改变Tile**    | 将地图上某Tile替换为另一Tile（如“墙变地板”“门消失”）。       | 目标坐标（x,y）、新Tile类型（地板）。             |
| **传送玩家**      | 将玩家传送到指定坐标/楼层（如“传送至2层(5,5)”）。           | 目标楼层（2）、坐标（x=5,y=5）。               |
| **显示对话**      | 在屏幕上显示文本对话（支持多行、角色头像）。                  | 对话文本（“老人：给你一把红钥匙！”）、头像（老人）。        |
| **显示提示**      | 屏幕顶部显示简短提示（如“获得红钥匙×1”）。                 | 提示文本（“获得红钥匙×1”）。                   |
| **触发自定义事件**   | 手动触发另一个自定义事件（用于逻辑复用）。                   | 自定义事件ID（如“event\_boss\_defeated”）。 |
| **设置自定义变量**   | 给自定义变量赋值（如“设置‘boss\_awake’=true”）。      | 变量名（boss\_awake）、值（true）。          |
| **播放音效**      | 播放指定音效文件（如“获得道具时播放‘get.wav’”）。          | 音效文件名（get.wav）。                    |
| **震动屏幕**      | 屏幕短暂震动（如“击败Boss时震动”）。                   | 震动强度（中）、持续时间（0.5秒）。                |
| **锁定/解锁玩家操作** | 禁止/允许玩家移动和操作（如“对话时锁定操作”）。               | 操作类型（锁定/解锁）。                       |
| **游戏胜利/失败**   | 直接触发游戏胜利或失败结局。                          | 结局类型（胜利/失败）、结局文本（“恭喜通关！”）。         |
| **无操作**       | 占位动作（用于条件分支，如“条件满足时获得道具，不满足时无操作”）。      | 无。                                 |

在原有JSON基础上，新增`events`字段（全局事件）和Tile的`events`字段（Tile事件）：

  "floors": [
    {
      "floorId": 1,
      "mapWidth": 20,

          "x": 5,

## 七、编辑器事件系统功能模块设计

### 1. 事件编辑面板（右侧抽屉式）

- **触发条件选择区**：下拉菜单选择触发类型，表单配置参数。
- **执行条件编辑区**：
  - “添加条件”按钮，支持多条件组合。
  - 可视化逻辑连接（“与”/“或”切换）。
  - 条件不满足提示文本输入框。
- **动作序列编排区**：
  - 拖拽式排序动作。
  - “添加动作”按钮，选择动作类型后配置参数。
  - 每个动作可设置“延迟执行时间”（毫秒）。
- **后续事件选择区**：下拉菜单选择已有的自定义事件。

### 2. 事件绑定方式

- **Tile绑定**：选中Tile后，右侧属性面板新增“事件”标签页，点击“添加事件”即可编辑。
- **全局事件**：顶部工具栏“事件”菜单 → “全局事件管理”，弹窗中添加/编辑全局事件。
- **自定义事件**：顶部工具栏“事件”菜单 → “自定义事件管理”，弹窗中添加/编辑自定义事件。

### 3. 事件模板库

内置10+常用事件模板，用户可一键应用后修改参数：

- 模板1：踩踏板开门（消耗钥匙 → 开门 → 提示）。
- 模板2：NPC对话给道具（对话 → 获得道具 → 标记事件已触发）。
- 模板3：击败怪物解锁隐藏墙（击败怪物 → 墙变地板 → 提示）。
- 模板4：属性不足无法通过（检查攻击力 → 不满足则显示提示并传送回原地）。

### 4. 事件可视化与验证

- **地图标记**：绑定事件的Tile显示特殊图标（如对话气泡、感叹号），点击图标可快速打开事件编辑面板。
- **事件逻辑检查**：
  - 自动检测“循环触发”（如事件A触发事件B，事件B又触发事件A）。
  - 自动检测“条件矛盾”（如要求“HP≥100且HP≤50”）。
  - 自动检测“动作无效”（如“消耗不存在的道具”）。
- **事件测试模式**：进入预览模式后，高亮显示所有事件Tile，触发事件时显示“事件执行日志”（如“触发事件A → 条件满足 → 执行动作1：获得道具”）。

## 八、技术实现要点

### 1. 前端状态管理

- 使用Redux Toolkit存储：
  - 当前编辑的事件列表（`currentEvents`）。
  - 事件模板库（`eventTemplates`）。
  - 事件验证结果（`eventValidationErrors`）。
- 事件编辑时，实时更新Redux状态，并同步到JSON预览区。

### 2. 事件逻辑解析（供游戏引擎参考）

游戏引擎加载JSON后，可通过以下步骤解析事件：

1. **监听触发条件**：如玩家移动时，检查目标Tile是否有“踩上Tile”事件。
2. **检查执行条件**：遍历事件的`conditions`数组，按`conditionLogic`判断是否满足。
3. **执行动作序列**：按顺序执行`actions`数组，支持异步动作（如显示对话后等待用户点击）。
4. **触发后续事件**：若`nextEvent`不为空，递归执行该自定义事件。

### 3. 编辑器UI组件库

- 使用React DnD实现动作序列的拖拽排序。
- 使用Ant Design的`Form`组件实现触发条件、执行条件的参数配置。
- 使用Monaco Editor（VS Code同款）实现JSON预览区的语法高亮和只读显示。

## 九、示例场景：“踩踏板开门”事件完整配置

### 场景描述

玩家在(10,10)踩上踏板Tile，消耗1把黄钥匙，开启(15,15)的黄门，显示提示“门开了！”。

### 配置步骤

1. **绑定Tile**：选中(10,10)的踏板Tile，添加事件。
2. **触发条件**：选择“玩家踩上Tile”。
3. **执行条件**：添加“钥匙数量”条件，配置为“黄钥匙≥1”，条件不满足提示“你需要黄钥匙！”。
4. **动作序列**：
   - 动作1：消耗钥匙（黄钥匙×1）。
   - 动作2：改变Tile（目标(15,15)，新Tile“地板”）。
   - 动作3：显示提示（“门开了！”）。
5. **后续事件**：无。
